Interactuar con AWS

En esta sección abordaremos dos aspectos fundamentales para comenzar a utilizar la nube de AWS:

  1. La creación de una cuenta en AWS para aprovechar sus servicios en la capa gratuita.
  2. Las formas de interactuar con los servicios de AWS: la consola de administración, el CLI y el SDK de AWS.

👤 Crea una cuenta en AWS​

En los siguientes pasos se describirá como crear una cuenta en AWS:

1. Acceda al sitio oficial de amazon para registrar una nueva cuenta: https://portal.aws.amazon.com/billing/signup#/start/email

2. Ingrese el correo electrónico del usuario raíz y el nombre de la cuenta.

Usuario raíz (Root user)

Esta es la identidad que otorga AWS cuando se crea por primera vez una cuenta. Se recomienda no utilizar el usuario raíz a nivel organizacional para tareas cotidianas ya que este usuario tiene acceso completo a todos los servicios y recursos de AWS en la cuenta.

Consejo para el nombre de la cuenta

Amazon nos recomienda seguir el estándar de denominación empresa-objetivo-entorno, por ejemplo: horizon-contabilidad-prod. En caso de que sea una cuenta personal, Amazon recomienda utilizar el estándar de denominación nombre-apellido-objetivo.

3. Verifique su correo electrónico ingresando un código de verificación enviado a su correo.

4. Ingrese una contraseña segura para su cuenta.

Restricciones para las contraseñas en AWS
  • La contraseña debe tener un mínimo de 8 caracteres y un máximo de 128 caracteres.
  • La contraseña debe incluir un mínimo de tres de los siguientes tipos de caracteres combinados: mayúsculas, minúsculas, números y símbolos ! @ # $ % ^ & * () <> [] {} | _+-=.

5. Ingrese su información de contacto empresarial como: el nombre completo, el nombre de la organización, el número de teléfono, el país o región, la dirección, la ciudad, la provincia y el código postal. Para consultar su código postal en una ubicación en Ecuador ingrese al sitio: https://www.codigopostal.gob.ec/. En este mismo apartado se pedirá que lea y acepte el contrato de usuario.

Asepctos importantes sobre el contrato del usuario
  • El usuario debe pagar por los servicios de AWS que utiliza, y AWS se reserva el derecho de suspender o cancelar el acceso a los servicios si no se realiza el pago.
  • AWS podrá aumentar o añadir nuevas tarifas y cargos por cualquier Servicio existente que usted utilice mediante aviso con 30 días de anticipación.
  • AWS podrá elegir cobrar un interés del 1,5 % mensual (o la tasa más alta permitida por ley, si fuese menor) sobre todos los pagos atrasados.

6. Luego ingrese los datos de su tarjeta para descontar termporalmente el valor de $1.00 USD para la activación de su cuenta en AWS.

7. Se validará otra información ingresada como el número de teléfono móvil por lo cual se pedirá que ingrese su número y un código de verificación.

8. Seleccione el plan de soporte que por defecto se marca en el plan gratuito.

9. Al finalizar, recibirá un correo electrónico de notificación de creación de una cuenta y podrá acceder a la página de inicio de su consola de administración.

🖥 Consola de administración de AWS​

La consola de administración es la aplicación web que provee Amazon para que un usuario pueda administrar sus recursos y pueda acceder a todos los servicios de AWS. Este panel de administración contiene diferentes elementos de gran utilidad como: el menú para acceder a todos los servicios de AWS, la barra de búsqueda, el CloudShell, las regiones en donde se desea operar y una opción para la configuración de la cuenta y la facturación de la misma.

Añadir un servicio a favoritos​

Al dar clic en la opción de menú de servicios, es posible acceder a todos los servicios de AWS por categoría y dentro de cada una seleccionar nuestros servicios favoritos. Esto se puede lograr dando clic en la parte izquiera del nombre del servicio en el ícono de favorito ⭐. Está opción es muy útil debido a que dispone un acceso directo y rápido de los servicios que el usuario más utiliza.

AWS CloudShell​

Es la una terminal que se incorpora en la plataforma de AWS para disponer de una CLI con herramientes preconfiguradas como el AWS CLI. Esta opción permite interactuar con los servicios de AWS directamente desde el navegador, y almacenar archivos o scriptes para un uso en sesiones futuras.

Regiones​

Según la documentación oficial de Amazon Web Service:

Las regiones son conjuntos de recursos de AWS situados en la misma zona geográfica. Para muchos servicios, puede elegir una Región de AWS que especifique dónde se administran los recursos y para otros servicios no es necesario como AWS Identity y Access Management.

Para establecer una zona es necesario ir a un servicio específico y seleccionar una zona en particular en la barra principal.


Recuerda

La mejor elección de una zona en AWS depende de varios aspectos como: la ubicación geográfica de los clientes de la organización, la latencia deseada, los requisitos de cumplimiento normativo y la disponibilidad de servicios específicos en una región determinada.

🧑🏻‍💻 Interfaz de línea de comandos (CLI) de AWS​

Es una herramienta open source para interactuar con los servicios de AWS a través de comandos en la terminal. La característica más importante del AWS CLI es que proporciona acceso directo a las API públicas de los servicios de AWS lo que permite al equipo de TI crear scripts de shell para administrar los recursos de la organización.

Instalación de la CLI de AWS en MacOS​

A continuación se describirán los pasos necesarios para instalar AWS CLI en una MacOS:

  1. Descargue el .pkg y coloque un nombre de salida con el modificador -o.
curl "https://awscli.amazonaws.com/AWSCLIV2.pkg" -o "AWSCLIV2.pkg"
  1. Instale el paquete con el software installer.
sudo installer -pkg ./AWSCLIV2.pkg -target /
  1. Para verificar que se ha instalado correctamente ejecute el comando aws --version
$ aws --version
➜ aws-cli/2.11.11 Python/3.11.2 Darwin/22.2.0 exe/x86_64 prompt/off

Instalación con Docker​

El AWS CLI puede también ser descargado a través de Amazon ECR Public el cual dispone del AWS CLI en una imagen oficial de Amazon, es decir, en un entorno basado en contenedores sin tener que preocuparse por la instalación. Para conocer más sobre esta opción es posible seguir los pasos que se encuentran en el artículo: "Uso de las imágenes oficiales Amazon ECR Public/Docker Hub de la AWS CLI versión 2".

🗄 Kits de desarrollo de software (SDK) de AWS​

Amazon ofrece SDKs para que los desarrolladores puedan disponer de librerías que les permitan interactuar con los servicios de AWS y sus recursos empleando los lenguajes de programación de su preferencia. Actualmente existen SDKs de AWS para lenguajes de programación y tecnologías como:

  • Java
  • Python
  • C++
  • Go
  • JavaScript
  • Kotlin
  • .NET
  • Node.js
  • PHP
  • Ruby
  • Rust
  • Swift

Como SDK de python existe la opción de Boto 3 que tiene compatibilidad nativa con las versiones 2.7+ y 3.4+ de Python. Una gran característica de Boto 3 es que incorpora “tareas en espera” que identifican automáticamente cambios en los estados predefinidos de los recursos de AWS.
